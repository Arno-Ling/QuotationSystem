## 项目概述

### 基本信息
- **项目名称**: 模具委外采购管理系统
- **架构模式**: Agent + Harness + Skills + RAG
- **开发阶段**: 基础设施搭建完成，进入核心开发阶段

### 项目目标
这是一个基于 **Agent + Harness + Skills + RAG** 架构的智能模具委外采购管理系统，旨在通过AI技术优化模具采购流程，并在关键决策点引入人机交互(HITL)机制。

**核心目标**:
1. **提高效率**: 自动化询价、报价、订单生成流程
2. **降低成本**: AI辅助决策，优化供应商选择
3. **质量保障**: 异常自动检测和智能定责
4. **可追溯性**: 完整的决策和操作记录
5. **人机协同**: 在关键节点保留人工决策权

### 技术栈

#### 后端
- **语言**: Python 3.9+
- **Web框架**: FastAPI
- **ORM**: SQLAlchemy
- **数据库**: MySQL 8.0+
- **AI框架**: LangChain
- **向量数据库**: ChromaDB / FAISS
- **认证**: JWT
- **缓存**: Redis

#### 前端
- **框架**: React 18
- **构建工具**: Vite
- **状态管理**: Redux Toolkit
- **UI组件**: Ant Design 5
- **样式**: Tailwind CSS
- **路由**: React Router v6
- **HTTP客户端**: Axios
- **拖拽**: react-beautiful-dnd

#### 基础设施
- **容器化**: Docker + Docker Compose
- **Web服务器**: Nginx
- **开发环境**: Windows/Linux/macOS

### 核心组件说明

#### 1. Harness (调度器)
**职责**:
- 管理所有Skills和Agent的注册
- 提供统一的请求入口
- 负责Skills和Agent的生命周期管理
- 状态查询和监控

**关键方法**:
- `register_skill(name, instance)` - 注册技能
- `register_agent(instance)` - 注册Agent
- `run_agent_request(request)` - 执行Agent请求
- `get_agent_state()` - 获取Agent状态

#### 2. Agent (智能代理)
**职责**:
- 业务流程编排核心
- 调用各种Skills完成具体任务
- 处理人机交互反馈
- 维护执行状态和历史

**关键方法**:
- `process_request(request)` - 处理业务请求
- `handle_human_feedback(feedback)` - 处理人工反馈
- `_call_skill(name, **kwargs)` - 调用技能
- `get_current_state()` - 获取当前状态

#### 3. Skills (技能模块)
**设计原则**:
- 独立的业务功能单元
- 可插拔设计
- 通过Harness注册和调用
- 继承BaseSkill基类

**核心Skills列表**:

| Skill名称 | 功能描述 | 关键方法 |
|----------|---------|---------|
| DataRetrievalSkill | 数据检索 | 查询零件、工艺、项目、供应商信息 |
| OutsourcingDecisionSkill | 委外决策 | 成本分析、MOQ判断、产能评估 |
| SupplierManagementSkill | 供应商管理 | 能力匹配、地理筛选、评级排序 |
| InquiryQuotationSkill | 询价报价 | 询价单生成、邮件发送、OCR解析 |
| TaskManagementSkill | 任务管理 | 任务创建、状态更新、订单生成 |
| ProgressTrackingAndKanbanSkill | 进度跟踪 | 看板数据聚合、状态统计 |
| ExceptionHandlingSkill | 异常处理 | 异常报告、AI定责、解决方案 |
| HumanInteractionSkill | 人机交互 | 审核任务创建、反馈记录 |
| RAGSkill | 知识检索 | 向量检索、历史案例查询 |

#### 4. RAG (检索增强生成)
**功能**:
- 从知识库检索相关信息
- 辅助AI决策
- 提供历史案例参考
- 支持多种查询类型

## 核心业务流程

### 主流程: 模具委外采购

```
开始
  ↓
1. 数据获取
  ├─ 获取零件信息 (材料、MOQ、成本)
  ├─ 获取工艺要求
  └─ 查询历史数据 (RAG)
  ↓
2. 委外决策 (AI分析)
  ├─ 成本对比分析
  ├─ MOQ判断
  ├─ 产能评估
  └─ 生成决策建议
  ↓
【人机交互点1: 委外决策确认】
  ├─ 项目经理审核
  ├─ 批准 → 继续
  ├─ 拒绝 → 内部生产
  └─ 修改 → 重新评估
  ↓
3. 供应商匹配
  ├─ 能力匹配
  ├─ 地理位置筛选
  └─ 评级排序
  ↓
4. 询价报价
  ├─ 生成询价单
  ├─ 发送邮件
  ├─ 接收报价 (OCR解析)
  └─ 数据入库
  ↓
【人机交互点2: 报价审核】
  ├─ 采购员审核
  ├─ 批准 → 生成订单
  ├─ 拒绝 → 重新询价
  └─ 谈判 → 修改价格
  ↓
5. 生成委外加工单
  ├─ 订单信息确认
  ├─ 保存到数据库
  └─ 通知相关人员
  ↓
6. 进度跟踪
  ├─ 看板可视化
  ├─ 状态实时更新
  └─ 异常监控
  ↓
7. 异常处理 (如有)
  ├─ 异常检测
  ├─ AI辅助定责
  ├─ 解决方案生成
  └─ 人工确认
  ↓
结束
```

### 人机交互(HITL)机制

系统在两个关键节点暂停自动执行，等待人工确认:

#### 交互点1: 委外决策确认
- **触发条件**: Agent建议委外
- **审核人**: 项目经理
- **操作选项**:
  - ✅ 批准 - 继续委外流程
  - ❌ 拒绝 - 转为内部生产
  - 🔄 修改 - 重新评估决策
- **数据记录**: human_review_tasks表

#### 交互点2: 报价审核
- **触发条件**: 收到供应商报价
- **审核人**: 采购员
- **操作选项**:
  - ✅ 批准 - 生成委外加工单
  - ❌ 拒绝 - 重新询价
  - 💬 谈判 - 修改价格后确认
- **数据记录**: human_review_tasks表

## 数据库设计

### 核心表结构 (11张表)

#### 1. users - 用户表
```sql
id, username, password_hash, role, email, phone
created_at, updated_at
```
**用途**: 用户认证和权限管理

#### 2. projects - 项目表
```sql
id, project_name, description, status
start_date, end_date, manager_id
created_at, updated_at
```
**状态流转**: 待报价 → 生产中 → 已交付 → 已完成

#### 3. molds - 模具表
```sql
id, project_id, mold_name, mold_number, status
created_at, updated_at
```
**关系**: 一个项目包含多个模具

#### 4. parts - 零件表
```sql
id, mold_id, part_name, part_number
material, quantity, moq, drawing_id, status
created_at, updated_at
```
**关系**: 一个模具包含多个零件

#### 5. suppliers - 供应商表
```sql
id, supplier_name, category, address
contact_person, contact_phone
capabilities (JSON), rating
created_at, updated_at
```
**用途**: 供应商信息和能力管理

#### 6. inquiries - 委外询价单
```sql
id, project_id, part_id, process_id
inquiry_date, due_date, quantity
technical_requirements, drawing_link, status
created_by, created_at, updated_at
```
**状态**: 待发送 → 已发送 → 已报价 → 已关闭

#### 7. quotations - 委外报价单
```sql
id, inquiry_id, supplier_id
quotation_date, unit_price, delivery_date
total_amount, remarks, is_assembly_included
quotation_file_path, status
created_at, updated_at
```
**状态**: 待审核 → 已接受 / 已拒绝

#### 8. outsourced_orders - 委外加工单
```sql
id, project_id, part_id, process_id
supplier_id, quotation_id
order_date, delivery_date, actual_delivery_date
quantity, unit_price, total_amount
is_assembly_included, status
created_by, created_at, updated_at
```
**状态**: 待生产 → 生产中 → 已完成 → 已入库 → 已结算

#### 9. exceptions - 异常单
```sql
id, project_id, related_entity_id, entity_type
exception_type, description
report_by, report_at
responsible_party, ai_analysis_report
evidence_package_path, resolution_plan
status, is_temporary_ignored
compensation_plan, due_date
created_at, updated_at
```
**状态**: 待处理 → 溯源中 → 待确认 → 已解决 / 暂缓处理

#### 10. tasks - 任务表
```sql
id, project_id, task_name, description
assigned_to, status, priority
start_date, due_date
actual_start_date, actual_end_date
created_at, updated_at
```
**用途**: 看板和流程管理

#### 11. human_review_tasks - 人机交互任务表
```sql
id, project_id, related_entity_id, entity_type
review_type, description, assigned_to
status, feedback_action, feedback_reason
feedback_by, feedback_at
created_at, updated_at
```
**审核类型**: outsourcing_decision, quotation_review, exception_resolution
**状态**: pending → approved / rejected / modification_requested

## 开发指南

### 添加新的Skill

1. 在 `backend/skills/` 创建新文件
2. 继承 `BaseSkill` 类
3. 实现 `execute()` 方法
4. 在 `main.py` 中注册

```python
from core.base_skill import BaseSkill

class MyNewSkill(BaseSkill):
    def __init__(self):
        super().__init__(
            name="MyNewSkill",
            description="我的新技能"
        )
    
    def execute(self, **kwargs):
        # 实现业务逻辑
```
