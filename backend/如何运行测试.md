# 如何自己运行测试

## 前提条件

1. **确保FastAPI应用正在运行**
   - 应用应该已经在运行（端口8000）
   - 如果没有运行，打开PowerShell，执行：
   ```powershell
   cd "C:\Users\Arno\Desktop\Quotation system\backend"
   python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

## 测试步骤

### 测试1: 检查API是否运行

打开浏览器，访问：
```
http://localhost:8000
```

应该看到：
```json
{
  "message": "AI Agent Backend API",
  "version": "1.0.0",
  "docs": "/docs",
  "health": "/health"
}
```

### 测试2: 查看API文档

打开浏览器，访问：
```
http://localhost:8000/docs
```

你会看到Swagger UI界面，可以直接在浏览器中测试API。

### 测试3: 测试异常分析端点（使用Python脚本）

**打开新的PowerShell窗口**（不要关闭运行应用的窗口），执行：

```powershell
cd "C:\Users\Arno\Desktop\Quotation system\backend"
python test_exception_endpoint.py
```

### 测试4: 测试异常分析端点（使用浏览器）

1. 打开浏览器访问：`http://localhost:8000/docs`
2. 找到 `POST /api/exception/analyze` 端点
3. 点击 "Try it out"
4. 使用以下测试数据（或修改为你自己的数据）：

```json
{
  "exception_id": "TEST001",
  "exception_type": "尺寸偏差",
  "description": "轴承座内径尺寸超差0.5mm，超出公差范围±0.2mm，影响装配精度",
  "related_entity_id": "PART001",
  "entity_type": "part",
  "project_id": "PROJ001",
  "supplier_id": "SUP001",
  "material": "钢",
  "process_type": "数控加工",
  "severity": "major",
  "quantity_affected": 50
}
```

5. 点击 "Execute"
6. 查看响应结果

### 测试5: 测试数据库连接

```powershell
cd "C:\Users\Arno\Desktop\Quotation system\backend"
python test_db_connection.py
```

这会告诉你：
- 数据库连接是否成功
- 数据库是否存在
- 表是否存在
- AI分析字段是否已添加

### 测试6: 使用curl测试（如果你喜欢命令行）

```powershell
# 测试根路径
curl http://localhost:8000/

# 测试健康检查
curl http://localhost:8000/health

# 测试异常分析（需要安装curl或使用PowerShell的Invoke-WebRequest）
Invoke-WebRequest -Uri "http://localhost:8000/api/exception/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "exception_id": "TEST001",
    "exception_type": "尺寸偏差",
    "description": "轴承座内径尺寸超差0.5mm",
    "related_entity_id": "PART001",
    "entity_type": "part",
    "project_id": "PROJ001"
  }'
```

## 常见问题

### Q1: 应用没有运行
**解决方案：**
```powershell
cd "C:\Users\Arno\Desktop\Quotation system\backend"
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### Q2: 端口8000被占用
**解决方案：**
```powershell
# 查找占用端口的进程
netstat -ano | findstr :8000

# 结束进程（替换<PID>为实际进程ID）
taskkill /PID <PID> /F

# 或者使用其他端口
python -m uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

### Q3: LLM API连接错误
**检查：**
1. `.env` 文件中的 `OPENAI_API_KEY` 是否正确
2. `.env` 文件中的 `OPENAI_BASE_URL` 是否正确
3. `.env` 文件中的 `EXCEPTION_AGENT_MODEL` 是否正确
4. 网络连接是否正常

### Q4: 数据库连接错误
**解决方案：**
1. 运行 `python test_db_connection.py` 诊断问题
2. 检查MySQL服务是否运行：`Get-Service MySQL*`
3. 检查 `.env` 文件中的 `DB_PASSWORD` 是否正确
4. 尝试用命令行连接：`mysql -u root -p`

## 成功的标志

### LLM API工作正常：
- 测试脚本返回 `"success": true`
- 有完整的分析报告
- `agent_steps` 大于1（说明Agent执行了多个步骤）

### 数据库工作正常：
- 没有 `database_update_warning` 字段
- 或者 `database_update_warning` 为 null

### 完全成功：
```json
{
  "success": true,
  "analysis_report": "【异常分析报告】...",
  "agent_steps": 8,
  "database_update_warning": null
}
```
